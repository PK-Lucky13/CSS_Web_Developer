Hierarquia do Efeito em Cascata

Estabelece uma prioridade para aplicação de uma regra de estilo, caso se tenha mais de uma regra para o mesmo seletor CSS.

Se houver conflito, o navegador irá:

  1. Analisar todas as regras;
  2. Ver o nível de importância na origem do elemento;
  3. Comparar a especificidade das declarações com mesmo nível de importância (entra o cálculo de especificidade);
  4. Em caso de empate do grau de especificidade, irá olhar a ordem de declaração colocada dentro dos arquivos de código. A regra CSS que for declarada por último, irá ser selecionada.
Ordem de hierarquia (com exemplos)
  1. index.html - Inline Tag: <div style="background-color: #FFF;">
  2. index.html - Tag HTML <style> .class {background-color: #000;} </style>
  3. Arquivo style.css - <link rel="stylesheet" href="assets/css/style.css">

BEM (Block, Element, Modifier)

B E M
  • Bloco
  • Elemento
  • Modificador
"Bem faz alusão a Programação Orientada a Objetos (POO), uma maneira de descrever a realidade no código, com uma variedade de padrões e uma menira de pensar nas entidades do programa." - Varga Stepanova, desenvolvedora front-end do projeto.

Bloco (Block)

  • Entidade indepentente (bloco de construção)
  • Elemento pai
  • [BLOCO] (.btn)

Elemento (Element)

  • Elemento filho
  • Não indepentente
  • Elemento Vinculado a um bloco
  • [BLOCO]__[ELEMENTO] (.btn__price_)

Modificador (Modifier)

  • Modifica um bloco ou elemento
  • Variante para alterar a aparência
    • Variar uma prioridade
    • Atribuir um estado
  • [BLOCO]__[ELEMENTO]--[MODIFICADOR]
    • .menu--dark
    • .btn--orange
    • menu__link--disabled
